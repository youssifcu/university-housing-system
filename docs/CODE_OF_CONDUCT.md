# 📋 CODE OF CONDUCT

## Our Commitment

We are committed to providing a welcoming and inspiring community for all. This Code of Conduct outlines expectations for participation in our community and procedures for addressing unacceptable behavior.

We pledge to make participation in this project a harassment-free experience for everyone, regardless of:
- Age
- Body size
- Disability
- Ethnicity
- Gender identity and expression
- Level of experience
- Nationality
- Personal appearance
- Race
- Religion
- Sexual identity and orientation

---

## 💪 Our Standards

### Positive Behaviors We Encourage

- **Be respectful** - Treat all community members with kindness and respect
- **Be inclusive** - Welcome people from diverse backgrounds and perspectives
- **Be patient** - Remember we're all learning and growing
- **Be constructive** - Offer helpful feedback to improve the project
- **Be collaborative** - Work together toward common goals
- **Be professional** - Keep interactions focused on the project
- **Give credit** - Acknowledge contributions and ideas from others
- **Ask questions** - It's okay to not know everything

### Unacceptable Behaviors

The following behaviors are NOT tolerated in our community:

- **Harassment** - Intimidation, threats, or discriminatory comments
- **Discrimination** - Unfair treatment based on personal characteristics
- **Abuse** - Disrespect, insults, or personal attacks
- **Trolling** - Deliberately provocative posts intended to upset others
- **Spam** - Unsolicited promotional content or repetitive messages
- **Doxxing** - Publishing private information without consent
- **Exclusion** - Deliberately excluding people or their ideas
- **Rudeness** - Unnecessarily hostile or aggressive communication

## 🚀 Project Participation Guidelines

### When Contributing Code

1. **Write clean code**
   - Follow style guides
   - Write descriptive commit messages
   - Add tests for new features
   - Document your changes

2. **Respect existing code**
   - Don't rewrite others' code without discussion
   - Ask before making major changes
   - Give credit to original authors

3. **Communicate openly**
   - Explain your approach to complex changes
   - Ask for code review
   - Respond to feedback constructively

4. **Test thoroughly**
   - Run tests before committing
   - Test on multiple devices/browsers
   - Document any known issues

### When Creating Issues

1. **Search first**
   - Check if issue already exists
   - Avoid duplicates

2. **Provide context**
   - Clear description of the problem
   - Steps to reproduce
   - Environment details (OS, version, etc.)
   - Expected vs actual behavior

3. **Be specific**
   - Not: "It doesn't work"
   - Yes: "Login button redirects to 404 instead of dashboard"

### When Creating Pull Requests

1. **Start from develop branch**
   - Create feature branch: `feature/description`
   - Keep scope manageable

2. **Write good commit messages**
   ```
   feat: add user authentication
   
   - Implement Firebase authentication
   - Add login and register screens
   - Handle token refresh
   
   Fixes #123
   ```

3. **Include description**
   - What does this PR do?
   - Why is this change needed?
   - How can reviewers test it?

4. **Be open to feedback**
   - Respond to all comments
   - Make requested changes
   - Discuss disagreements politely

### When Reviewing Code

1. **Be constructive**
   - Focus on the code, not the person
   - Suggest improvements, don't demand
   - Ask questions to understand reasoning

2. **Be thorough**
   - Check functionality
   - Review for bugs
   - Verify it follows guidelines
   - Test locally if possible

3. **Be timely**
   - Review in reasonable time
   - Don't leave PRs hanging
   - Approve when satisfied

## 🤝 Community Interactions

### In Issues & PRs

✅ **Do:**
- "I think we could improve this by..."
- "This is great! One suggestion: ..."
- "I'm not sure I understand. Can you explain...?"
- "Thanks for fixing this! Quick question..."

❌ **Don't:**
- "This is wrong"
- "You're doing it all wrong"
- "This is bad code"
- "Why would you do it that way?"

### In Discussions

✅ **Do:**
- Share experiences and knowledge
- Ask for help respectfully
- Provide helpful answers
- Celebrate others' achievements

❌ **Don't:**
- Gatekeep information
- Make assumptions about others
- Post off-topic promotions
- Share others' private information

## 📞 Reporting Issues

If you witness or experience unacceptable behavior:

### Step 1: Assess Safety
- If anyone is in immediate danger: **Call emergency services**

### Step 2: Document
- Note what happened
- Record time and date
- Save relevant messages
- Note witnesses

### Step 3: Report

**For online harassment:**
- Email: conduct@university-housing-system.com
- Include date, time, and description
- Provide screenshots if applicable

**For in-person incidents:**
- Notify project maintainers immediately
- Email conduct@university-housing-system.com

### Step 4: Investigation

We will:
- Acknowledge receipt within 24 hours
- Investigate thoroughly and fairly
- Maintain confidentiality as much as possible
- Take appropriate action

## ⚖️ Consequences

Community members who violate this Code of Conduct may face:

1. **Warning** - For minor, first-time violations
   - Private discussion about behavior
   - Request for improvement
   - Monitoring of future interactions

2. **Temporary Ban** - For repeated or serious violations
   - Removal from repository for 7-30 days
   - Cannot comment on issues/PRs
   - Cannot merge code

3. **Permanent Ban** - For severe or repeated violations
   - Removed from all project spaces
   - Cannot contribute
   - Reported violations may be escalated

**Note:** Consequences are proportional to the violation severity and intent.
